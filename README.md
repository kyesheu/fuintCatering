## 🎯 项目简介

fuint 是一套面向**实体门店**的**企业级开源会员营销与收银管理系统**，基于前后端分离架构，后端采用 **Java SpringBoot + MyBatis-Plus + MySQL + Redis**，前端覆盖 **Uniapp（H5 / 微信小程序 / App）+ Element UI（管理后台）+ Electron（桌面收银台）** 三端矩阵。

系统以「线下私域流量运营」为核心设计理念，打通收银系统与会员营销之间的壁垒——**支付即成为会员**，顾客扫码消费自动进入商家私域流量池。通过积分体系、会员等级、卡券系统（优惠券 / 储值卡 / 集次卡）、消息推送等工具，帮助商户降低获客成本、提升复购率。

## 💡 业务特色

### 1. 会员运营自动化
商家通过日常活动设置（开卡礼、沉睡唤醒等），顾客成为会员后自动赠送优惠券，提升会员黏性和运营效率。

### 2. 收银 + 会员一体化
打通收银系统与会员营销的壁垒。代客下单收银，支付即成为会员，线上线下统一渠道。

### 3. 完整的会员权益体系
积分兑换、积分转赠、会员等级权益、积分加速、买单折扣，构建多层次会员激励。

### 4. 卡券矩阵齐全
储值卡、电子券、优惠券、集次卡、计次卡、实体卡购买与兑换、会员充值、余额支付。

### 5. 智能收银系统
后台管理员可帮助临柜会员下单、扫码支付，对接扫码枪、扫码盒子、云打印机，订单提交后自动打印小票。

### 6. 多通道消息触达
支持手机短信、站内弹框消息、微信订阅消息——发货消息、卡券到期提醒、活动提醒、会员到期提醒、积分余额变动提醒等。

## Ehcache + Redis 双重缓存体系

针对餐饮场景「高并发读、低频率写」的特点，系统设计了 L1 + L2 两级缓存架构：

| 维度 | Ehcache (L1 — 本地) | Redis (L2 — 分布式) |
|------|---------------------|----------------------|
| **部署方式** | JVM 堆内（进程内） | 独立进程（远程） |
| **访问延迟** | 纳秒级 | 毫秒级 |
| **存储容量** | 200,000 条目 | 受服务器内存限制 |
| **典型应用** | 商品列表、分类树、枚举值 | Session 共享、Token 认证、分布式锁 |
| **TTL 策略** | defaultCache 120s / store 300s | Session 7天 / 验证码 30min / 微信Token 2h |
| **一致性** | 实例间弱一致（TTL 兜底） | 全局强一致 |
| **序列化** | Kryo（高性能二进制） | Jackson2 JSON |
| **故障影响** | 实例重启丢失（可溢写磁盘恢复） | 独立于应用生命周期 |

### Redis 五大核心职责

```
┌─────────────────────────────────────────────────────┐
│  Redis 在 fuint 中的五大应用场景                       │
│                                                     │
│  ① Session 共享  ← @EnableRedisHttpSession          │
│  ② Token 认证    ← FOOD_USER_{token} / FOOD_ADMIN_  │
│  ③ 分布式锁      ← SET NX EX + Lua 原子解锁          │
│  ④ 验证码存储    ← 30分钟 TTL                        │
│  ⑤ 微信Token缓存 ← 2小时 TTL                         │
└─────────────────────────────────────────────────────┘
```

## 多端协同：核心业务流向

### 桌码点单全链路（顾客扫码 → 下单 → 支付 → 收银端感知）

```
顾客扫码桌台二维码
     │
     ▼
Uniapp 解析 URL: ?tableId=520&storeId=12&merchantNo=10001
     │
     ├── tableId → localStorage → 所有 API 请求 Header 自动携带
     └── 跳转 pages/category/index（点单页）
     │
     ▼
后端 ClientCartController / ClientOrderController
  ├── 根据 tableId 关联桌台 (mt_cart.TABLE_ID)
  ├── 根据 storeId 加载店铺商品
  └── 下单时写入 mt_order.TABLE_ID
     │
     ▼
支付成功 → mt_table 状态更新为 OCCUPIED
     │
     ▼
Cashier 端轮询 getTableList() → 实时感知桌台状态变更
```

### 收银核销流程（扫码枪 → 后端验证 → 核销记录）

```
收银员扫描顾客会员码 / 券码
     │
     ▼
POST /merchantApi/order/confirm
     │
     ▼
后端校验链：
  ✅ 卡券未过期  ✅ 未核销  ✅ 适用店铺匹配
  ✅ 预存卡余额扣减 / 集次卡次数扣减 / 优惠券状态更新
     │
     ▼
写入：mt_confirm_log + mt_user_coupon + mt_balance
```

```bash

    --spring.datasource.url=jdbc:mysql://localhost:3306/fuint-food \
    --spring.datasource.username=root \
    --spring.datasource.password=yourpassword \
    --spring.redis.host=localhost \
    --spring.redis.port=6379
```

## 📖 使用说明

### 前台（会员端）

1. **会员登录**：手机号 / 微信授权登录，查看卡券列表和会员权益。
2. **扫码点单**：扫描桌台二维码进入点单页，选择商品 → 加入购物车 → 确认就餐人数 → 提交订单 → 在线支付。
3. **卡券中心**：领取优惠券、购买预存卡、集次卡使用记录。
4. **卡券核销**：会员出示二维码，管理员扫描即可核销。
5. **卡券转赠**：将卡券转赠给其他会员（输入手机号），对方收到短信通知。
6. **积分与余额**：查看积分明细、余额变动、在线充值。
